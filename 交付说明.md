## 🎯 三个核心场景

### 场景一：评委打分
1. 访问首页 → 输入账号密码 → 登录
2. 自动跳转 `/scoring` 评委打分页
3. 看到自己负责的维度（其他维度不显示）
4. 每个项目依次打分（0-满分）+ 评论（选填）
5. 提交即保存，可重复进入修改
6. 截止日期：2026-06-30

### 场景二：管理员管理（admin登录）
1. 登录后跳 `/admin`
2. 功能：
   - 查看所有项目评分进度
   - 添加/编辑/删除项目
   - 创建新评审会（可复制历史项目模板）
   - 重置某项目/某评委的评分
   - 导出Excel/JSON全量数据
   - **一键生成报告HTML**（跳到 `/report` 页）

### 场景三：报告生成
1. admin点击"生成报告" → 跳 `/report`
2. 自动汇总：
   - 项目总分排名
   - 各项目问题清单（problems[]）
   - 各项目动作清单（actions[]）
   - 评委贡献度
   - 评分完成率
3. 一键下载HTML报告（可在飞书分享）

## ⚠️ 注意事项

### 评委维度限制
- W评5个维度（满分100）
- N/S评3个维度（满分70）
- J/G评3个维度（满分45）
- 完成率基准：17条维度配置（每位评委负责维度总和）

### 截止日期
- 数据库当前会议deadline=2026-06-30
- 超期后评委无法修改评分，管理员可解锁

### 数据隔离
- 评委只能看/改自己的评分
- 评分按 (meeting_id, project_id, reviewer_code, dim_name) 唯一约束
- 重复提交自动upsert覆盖

## 🔄 后续操作

### 评委打分流程
1. 把URL和账号密码发给5位评委
2. 评委在6-30之前完成打分
3. 你以admin登录看进度
4. 截止后生成报告HTML分享给领导

### 新建评审会
1. admin登录 → /admin
2. 点"新建评审会"
3. 可选"复制上次项目模板"
4. 截止日期至少给评委留一周

### 修改截止日期
直接改Supabase meetings表的deadline字段

## 🆘 故障排查

| 问题 | 解决方案 |
|---|---|
| 登录提示"账号不存在" | 账号必须大写（W/N/S/J/G/admin） |
| 登录提示"密码错误" | 密码见上方表格 |
| 评委看不到某个维度 | 检查 reviewer_dims 表该评委配置 |
| 提交评分报"已超过截止日期" | admin改meetings.deadline |
| 报告数据不对 | 检查scores表的reviewer_code是否正确 |
